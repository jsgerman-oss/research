# RC-05 — Plan-cache replay harness (design, no live API)

## Goal
Build the **agentic-plan-cache replay harness** that §7 "Plan-cache
evaluation" leaves as a `\tothink`. The harness extracts plan signatures
from historical Blackrim dispatches, builds an index, and on replay
predicts cache hit/miss + score against gold (the historical actual
plan). Acceptance bar from §7: cache hit rate ≥30%, false-hit rate
<5%. This phase ships the harness without the cross-validation run —
that's RC-06.

## Acceptance criteria
- `scripts/eval-plancache/{signature,build_index,replay}.py` exist.
- `scripts/eval-plancache/plan-index.jsonl` has ≥30 dispatch
  records.
- `data/aggregated/plancache-eval.csv` exists with per-dispatch
  rows.
- `data/aggregated/plancache-summary.csv` exists with
  hit_rate + false_hit_rate.
- §7 not yet updated (RC-06 does that with the final numbers).

## Failure modes
- Dispatch data not where expected: investigate alternatives
  (session transcripts, bd JSON export, telemetry replay). If
  ultimately unobtainable, downscope to ≥10 dispatches with
  manual reconstruction OR weaken the §7 claim to "preliminary
  evidence". Don't fabricate.
- All plan signatures collapse to the same hash (abstraction too
  coarse): raise the granularity (include more arg detail) and
  re-run.
- All signatures unique (abstraction too fine): lower granularity.
  This is the central design tension — document where you landed.
