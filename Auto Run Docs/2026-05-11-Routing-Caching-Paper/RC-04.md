# RC-04 — Routing eval: implement semantic-similarity router + live execution

## Goal
Implement the proposed semantic-similarity router, run it against the
50-turn test set from RC-03, produce
`data/aggregated/routing-eval-semantic-similarity.csv`, and write §7's
routing-evaluation paragraph with the measured numbers.

## Context — read first
- Paper dir: `/Users/jayse/research/blackrim-routing-caching-paper/`
- RC-03 must be complete. Test set + harness must exist.
- The proposed router is described in
  `/Users/jayse/Code/blackrim/docs/research/tier-down-main-thread-landscape.md`
  Section "Phase A recommendation: semantic-similarity-router". Read
  it BEFORE implementing — the design pre-commits to: (a) embedding
  the turn text via a small open model, (b) nearest-neighbor against
  a hand-curated set of tier-labelled exemplar turns, (c)
  K=5 with majority vote, (d) confidence threshold for opus-escalate.
- Acceptance bar from §7: precision ≥0.90 overall, recall ≥0.95 on
  the "should-be-opus" class (the costly-mistake direction).

## Budget + safety
- Embedding inference: if using a local SentenceTransformers model
  (e.g. `all-MiniLM-L6-v2`), zero API cost.
- If the router needs to call Claude API for any reason (it
  shouldn't — that defeats the purpose), STOP and ask.
- Total expected cost: ~$0 if local embeddings; eval grading
  is pure CSV comparison.

## Tasks
- [ ] Build the exemplar set. From the 50 hand-labelled turns,
  hold out 10 as test-time (write them to
  `scripts/eval-routing/test-set.yml`) and use the remaining 40 as
  exemplars (`scripts/eval-routing/exemplars.yml`). The test/exemplar
  split must be stratified by gold tier (don't leave all opus turns
  in test set).
- [ ] Implement `scripts/eval-routing/routers/semantic_similarity.py`:
  - Load exemplars at init; embed each exemplar turn text via
    SentenceTransformers `all-MiniLM-L6-v2` (or equivalent local
    model — pin the model id in code + README).
  - On `route(turn_text)`: embed the input, find top-K=5 nearest
    exemplars by cosine similarity, majority-vote the tier.
  - Confidence threshold: if top-5 majority is <0.6 (i.e. <3
    agree), escalate to opus. This implements the "wrongly cautious"
    bias from RC-03.
  - Cache embeddings in `scripts/eval-routing/.embedding-cache/`
    (gitignore this dir) so re-runs are fast.
- [ ] Wire the router into `run.py` (RC-03 already had the
  `--router semantic-similarity` flag stubbed).
- [ ] Run the eval on the held-out 10:
  `python scripts/eval-routing/run.py --router semantic-similarity
   --turns scripts/eval-routing/test-set.yml
   --out data/aggregated/routing-eval-semantic-similarity.csv`.
  Note: this runs against the 10-turn TEST set, not all 50. The
  exemplar set was used to fit the router; using it in eval is
  contamination.
- [ ] Inspect the per-tier metrics in `routing-summary-semantic-
  similarity.csv`. Check:
  - Overall accuracy ≥0.90
  - Recall on opus class ≥0.95
  - cost_saved_vs_opus > 0 (otherwise the router is worse than
    always-opus)
  If acceptance fails, DO NOT tune the router on the test set.
  Report the failure honestly and treat in §7 as a limitation.
- [ ] Rewrite §7 routing-evaluation paragraph
  (`sections/07-evaluation.tex` lines 58-66). Drop `\tothink`.
  New prose: setup (10-turn test set, stratified, frozen),
  baselines (always-opus / random from RC-03), proposed router
  (semantic-similarity with the measured numbers), interpretation
  (does it hit the acceptance bar? what's the cost saving on this
  small test set? what's the latency overhead?). Embed a small
  table `\label{tab:routing-results}` with per-router rows.
- [ ] Update §1 contribution claim if the routing result doesn't
  hit the bar. The paper currently promises "35-50% main-thread
  cost reduction at <1% quality loss". If measured is lower or
  quality loss higher, soften the claim in §0 + §1 + §7. Same
  non-negotiable rule as TRIM-04: measurement wins.
- [ ] Commit: `eval(rc): semantic-similarity router implementation
  + measured eval on 10-turn test set`.

## Failure modes
- SentenceTransformers won't load (no internet / model not cached):
  pre-stage the model. `pip install sentence-transformers` then
  `python -c "from sentence_transformers import SentenceTransformer;
  SentenceTransformer('all-MiniLM-L6-v2')"` once to cache.
- Test set too small to be statistically meaningful: it is. Report
  per-turn results in the appendix; treat the 10-turn aggregate as
  illustrative, not significance-tested. Flag in §7 limitations.
- Router collapses to "always-opus" (escalation threshold too
  tight): tune the threshold ONCE based on this measurement,
  rerun, and document the tuning in the §7 prose. If you find
  yourself tuning multiple times against the same test set,
  that's overfitting — pick the first reasonable threshold and
  publish.
