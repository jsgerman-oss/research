# RC-02 — Cost-split telemetry: backing the 99.4% main-thread number

## Goal
Back the paper's headline empirical claim — 99.4% main-thread / 0.6%
dispatch cost share (currently in `data/raw/session-evidence.json` and
in §7 Table 1) — with a **reproducible aggregation** from raw telemetry,
not a screenshot. Produce `data/aggregated/cost-split.csv` and reconcile
§7.

## Tasks
- [ ] Verify the pricing TOML exists. Check structure:
  `head -30 /Users/jayse/Code/blackrim/internal/pricing/sheets/anthropic-public.toml`.
  Expected: per-model {input_usd_per_mtok, output_usd_per_mtok,
  cache_creation_usd_per_mtok, cache_read_usd_per_mtok}.
- [ ] Create `scripts/aggregate-cost-split.py`. Reads the JSON-lines
  pull (from `data/raw/session-telemetry.json` or `--telemetry path`)
  + the pricing TOML (`--pricing path`). For each record, computes
  `cost_usd = (input_tokens × input_rate + output_tokens × output_rate
   + cache_creation × cc_rate + cache_read × cr_rate) / 1e6`.
  Partitions records by `source` field:
  - `source == "main-thread"` or `agent == "claude-code-main"` → main-thread
  - any subagent dispatch source → dispatch
  - unknown / missing source → unclassified (flag count in stderr;
    must be <5% of total)
  Emits CSV with columns: `bucket, cost_usd, n_calls, cost_share_pct`.
  Buckets: `total, main_thread, dispatch, unclassified`.
- [ ] Run it:
  `python scripts/aggregate-cost-split.py
   --telemetry data/raw/session-telemetry.json
   --pricing /Users/jayse/Code/blackrim/internal/pricing/sheets/anthropic-public.toml
   > data/aggregated/cost-split.csv`.

## Acceptance criteria
- `scripts/aggregate-cost-split.py` exists and is idempotent.
- `data/aggregated/cost-split.csv` exists with 4 bucket rows.
- §7 Table 1 cites the CSV, not the screenshot, as primary source.
- Unclassified bucket is <5% of total.

## Failure modes
- If `source` field is missing on >5% of records, the partitioning
  is unsound. STOP. Either fix the upstream telemetry (out of scope
  of this paper) or weaken the §7 headline number to a confidence
  interval. Don't fudge.
- If the pricing TOML is missing fields for some models in the
  invocations, log the gaps in stderr and use the nearest-tier
  fallback (haiku→sonnet→opus rates). Document fallbacks used in
  the commit body.

## Out of scope
- Per-model breakdown (could be a future enhancement; not needed
  for the headline 99.4% claim).
- Time-series view (would be useful for showing trend, but §7
  currently makes a single-window claim).
