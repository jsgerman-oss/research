## Acceptance criteria

## Failure modes
- Hit rate < 30%: this is a real result. Don't lower the threshold
  to pass; instead, write §7 honestly and §8 discusses what would
  need to change (more dispatches, finer signature, semantic
  similarity beyond signature hashing).
- False-hit rate > 5%: a worse failure than low hit-rate, because
  false-hits actively degrade correctness. Investigate which plans
  are colliding incorrectly — likely a too-coarse signature.
  Document in §7. Don't tune away the false-hits silently.
- Composed cost reduction less than just prefix-cache: plan-cache
  is providing zero or negative marginal value on this dataset.
  Don't claim 50% savings; report what was measured.
