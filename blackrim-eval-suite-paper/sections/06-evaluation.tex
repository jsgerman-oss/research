\section{Empirical Evaluation}
\label{sec:eval}

\tothink{full empirical-evaluation prose; every quantitative claim
deferred until paper-stream telemetry has accumulated and per-judge
labelled calibration sets are built. The headline finding, when the
data lands, is the RLHF-degrades-judge-calibration result --- compare
ECE-base-model vs ECE-RLHF-model on the same scorer template across
$\geq 200$ cases per judge.}

\subsection{Setup}

\paragraph{Trace.}
Per-case telemetry from the planned
\texttt{internal/eval/paper\_stream.go} flowing to
\texttt{data/raw/cases.jsonl}: case ID, suite ID, judge model,
rubric ID, judge log-probs over the choice set, judge rationale,
ground-truth label (where applicable), inferred (scorer-class,
task-shape) cell, latency. Trace window: \tothink{30}-day rolling.
Total cases: \tothink{N}; per-cell breakdown
\tothink{$N_{c_1}, \ldots, N_{c_{24}}$}.

\paragraph{Calibration sets.}
Per-judge-model labelled calibration sets at
\texttt{evals/\_calibration/<judge>/cases.jsonl}: \tothink{200--500}
cases per judge, human-labelled by domain experts, refreshed every
\tothink{30} days per OQ-E7. Used to fit per-(judge, template)
temperature scalars and as the held-out reference for the headline
calibration finding.

\paragraph{Baselines.}
\begin{itemize}[leftmargin=*]
  \item \textbf{Argmax-decoded LLM classifier} --- the current Blackrim
        default; the literature's typical "LLM-as-judge" baseline.
  \item \textbf{No-temperature LLM classifier} --- probability-weighted
        but $T = 1$ (no calibration correction). Isolates the
        Layer-1 contribution from Layer-3.
  \item \textbf{Probability-weighted + temperature-scaled} (ours,
        Layers 1+3). The proposed v1 calibration-axis configuration.
  \item \textbf{Mean-of-means aggregator} --- the current gate.
  \item \textbf{Three-axis gate} (ours) --- per-cell calibration /
        agreement / bias-bound triple per \cref{sec:problem}.
\end{itemize}

across the available Anthropic model line --- pretrained-base if

judges have $\ECE \approx 0.13$--$0.25$ even on tasks where their
pretrained ancestors were near-zero. \tothink{this is the load-bearing

\subsection{Ablation: removing temperature scaling}

\tothink{ablate Layer 3. Expected finding: ECE on RLHF-tuned judge
cells (C.5--C.7) reverts from $\sim 0.05$ to $\sim 0.15$, breaking
calibration-no-regression on every cell that had been brought into
compliance by the scaling. Quantifies the load-bearing role of
temperature scaling --- not a "nice-to-have," a "necessary-for-the-frame."}

\subsection{Reproducibility}
\label{sec:repro}

Every quantitative claim is backed by a CSV in \texttt{data/} produced
by a script in \texttt{scripts/}. The pipeline:

\begin{lstlisting}[language=bash]
python scripts/pull-telemetry.py --since=30d > data/raw/cases.jsonl
python scripts/aggregate-by-cell.py < data/raw/cases.jsonl \
    > data/aggregated/by-cell.csv
python scripts/run-calibration.py --judge=haiku-4-5 \
    > data/aggregated/calibration-haiku.csv
python scripts/run-calibration.py --judge=sonnet-4-6 \
    > data/aggregated/calibration-sonnet.csv
python scripts/run-bias-diagnostic.py     > data/aggregated/bias.csv
python scripts/run-three-axis-gate.py     > data/aggregated/three-axis-gate.csv
make            # latex picks up new data/*.csv automatically via pgfplots
\end{lstlisting}

The pipeline is deterministic given fixed RNG seeds and a snapshot of
the production telemetry plus per-judge calibration sets; we anchor
the published version to a tagged commit. The companion artefacts
repository \cite{blackrim_evalsuite_paper_repo} hosts the data and
scripts; the calibration sets themselves are versioned alongside the
suites under \texttt{evals/\_calibration/}.
